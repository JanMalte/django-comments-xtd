django-comments-xtd is a reusable Django app that extends django-contrib-comments with thread support, following-up notifications and comments that only hit the database after users confirm them by email.

Threaded comments

Comments can be nested. The maximum thread level is customizable for the whole site and explicitly for each app.model pair.

Notifications and mute-links

Comment authors can receive notifications of follow-up comments. Notifications can be cancelled by clicking on the mute-link.

Comment confirmation

Comment confirmation via email when comment authors are not authenticated. Comments hit the database once they are confirmed.

Comment reactions

Starting with like and dislike reactions, they can be extended to any set of reactions to suit the comments of your website.

Object reactions

Reactions posted to the object being commented: story, photo, shop article... Integrate them with the comment form or separared.

CSS and JavaScript

It uses a full responsive CSS layout and vanilla JavaScript. It does not depend on any frontend framework.

Comments in Action

Three different apps use django-comments-xtd in this project in different ways. The stories app, the quotes app, and a shop app.

Stories app

Comments can be nested two levels. Reactions have been extended. Both, comments and reactions are sent using vanilla JavaScript.

Go to the stories page.

Quotes app

The quotes app works as if the user had disabled JavaScript in the browser. Everything works as in the stories app, but it's fully driven by the backend.

Go to the quotes page.

Shop app

The shop illustrates how to use 'object reactions' associated with comments. Users can send a product evaluation and a comment at the same time.

Go to the shop page.
